# Pitch: Accent Tint Matrix — независимый ревью

## Контекст

`lab-ui` — это zero-runtime генератор дизайн-токенов. На входе YAML с примитивами (neutral, accent), на выходе — статический CSS/JSON. Цвета строятся в CAM16-UCS, а не в sRGB, чтобы шкалы воспринимались как равномерные. Ветка `feature/accent-tint-matrix` добавляет генерацию «тинтов» — спектров примешивания акцента к нейтральному фону.

## Проблема, которую решаем

В Figma тинты описаны как прозрачный оверлей: `Brand@72` = базовый акцент с opacity 72% поверх фона. Это не переносится в CSS напрямую: один и тот же `rgba(..., 0.72)` даёт разный результат на `#FFFFFF` и на `#101012`. Нам нужны **сплошные hex**, предвычисленные индивидуально для каждой пары (акцент, фон), сохраняя перцептивную равномерность.

## Что в PR

### 1. Core: `tint.rs` + `accent.rs`

- `perceptual_mix(fg, bg, strength)` — линейная интерполяция в CAM16-UCS (`J'a'b'`), а не в sRGB. Это принципиально: sRGB midpoint красного и синего даёт `#800080`, CAM16-UCS даёт другой оттенок, воспринимаемый как истинная середина.
- `AccentConfig` — поддержка одного hex на все темы или отдельных значений для light/dark/ic_light/ic_dark. Паттерн заимствован у iOS System Colors: тёмная тема использует более светлый/насыщенный акцент.
- `create_accent_tint_matrix(accent_base, neutral_scale, strengths)` — возвращает матрицу `[strength_index][bg_step]`. Для 8 ступеней крепости и 13 шагов neutral получаем 104 hex на акцент на тему.

### 2. CLI: генерация CSS

`main.rs` расширен: читает секции `accents` и `tint` из конфига, строит 4 блока (`:root`, `.dark`, `.ic`, `.dark.ic`), выводит:
- `--accent-{name}` — чистый акцент
- `--tint-{name}-{strength}-on-neutral-{step}` — предвычисленный mix

JSON-вывод содержит те же ключи для программного доступа.

### 3. Схема и конфиг

`tokens.schema.json` обновлён: `accents` принимает либо строку-hex, либо объект с 4 темами. `tint.strengths` — массив процентов (дефолт: `[2, 4, 8, 12, 20, 32, 52, 72]`, извлечено из Figma).

## Философия и ограничения

- **Zero runtime deps**: всё предвычислено. В браузер уходит только CSS-переменные, никакого WASM/JS color-math.
- **Compact config**: пользователь указывает один hex, алгоритм порождает 416 значений (104×4 темы).
- **Не копируем Figma**: значения не скопированы из Figma 1:1. Извлечён принцип (8 ступеней крепости на 13 фонах), реализация — через CAM16-UCS mix.

## Что НЕ вошло (намеренно)

- **Semantic layer**: токены вроде `text-accent-primary` — отдельный PR после реструктуризации в Figma.
- **Neutral derivables**: алгоритмическая генерация neutral из одного hex — будущая задача, сейчас neutral задаётся явно.

## На что смотреть ревьюеру

1. **Корректность CAM16-UCS**: достаточно ли `J'a'b'` линейной интерполяции для воспринимаемой равномерности? Есть ли edge-case при смешивании на границах шкалы (белый/чёрный)?
2. **Конфиг и ошибки**: `AccentValue` — untagged enum в serde. Как ведёт себя десериализация при частичных полях (например, только `light` без `dark`)?
3. **Производительность**: матрица 104×4 = 416 вызовов `perceptual_mix` на акцент. На практике — мгновенно, но стоит ли кэшировать при масштабе >20 акцентов?
4. **Нейминг CSS**: `--tint-brand-20-on-neutral-7` — читаемо? Альтернатива: `--brand-tint-20-bg-7`?
